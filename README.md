# Лабораторная работа: синтаксический анализатор на Flex и GNU Bison

## Теоретическая справка

Перед семантической обработкой программы выполняются два шага:

- **Лексический анализ**: исходный текст разбивается на токены.
- **Синтаксический анализ**: последовательность токенов проверяется на соответствие грамматике.

В проекте используется стандартная связка:

- **Flex** генерирует лексер `yylex()` по файлу `mini.l`.
- **GNU Bison** генерирует парсер `yyparse()` по файлу `mini.y`.

Лексер возвращает токены парсеру, парсер проверяет корректность структуры и выполняет действия грамматики.

Инструкции по установке инструментов: [docs/bison-flex-install.md](docs/bison-flex-install.md).

---

## Функциональность

Реализован распознаватель мини-языка присваивания полям структуры с арифметическими выражениями.

Поддерживается:

- доступ к полям структуры: `a.b.c`
- операции: `+`, `-`, `*`, `/`
- круглые скобки
- приоритет `*` и `/` выше `+` и `-`
- левая ассоциативность бинарных операций
- ведущий унарный минус: `x.y = -2.5;`

Ограничения:

- унарный минус допускается только в начале `<Expr>`
- отсутствуют функции, сравнения и логические операции

## Грамматика

```bnf
<Program> ::= <StmtList>
<StmtList> ::= <Stmt> | <Stmt> <StmtList>
<Stmt> ::= id <FieldTail> "=" <Expr> ";"
<FieldTail> ::= "." id <FieldTail> | ε
<Expr> ::= "-" <Sum> | <Sum>
<Sum> ::= <Prod> | <Sum> "+" <Prod> | <Sum> "-" <Prod>
<Prod> ::= <Primary> | <Prod> "*" <Primary> | <Prod> "/" <Primary>
<Primary> ::= "(" <Expr> ")" | NUMBER | id <FieldTail>
```

---

## Терминалы

- `id` — идентификатор, начинающийся с буквы
- `NUMBER` — вещественное число (`double`), включая экспоненциальную форму
- символы: `"="`, `"."`, `"+"`, `"-"`, `"*"`, `"/"`, `"("`, `")"`, `";"`

---

## Примеры

Корректные:

```txt
a.b = 1 + 2;
a.cd.efg = a.b + 3;
root.node = -(1 + a.b);
x.y = -2.5;
data.value = (a.b + 2) * 3;
```

Некорректные:

```txt
examples/invalid/bad.mini               // два подряд "."
examples/invalid/bad_identifier.mini    // неверный id
examples/invalid/bad_unary_minus.mini   // унарный минус не допускается внутри Prod
examples/invalid/bad_missing_expr.mini  // отсутствует выражение
```

## Структура каталога

- `CMakeLists.txt` — сборка через CMake
- `files/mini.l` — лексер Flex
- `files/mini.y` — грамматика Bison и действия
- `main.cpp` — входная точка (чтение файла из аргумента)
- `examples/valid/` — корректные программы
- `examples/invalid/` — программы с ошибками
- `examples/run_tests.sh` — скрипт автопроверки
- `docker/` — окружение для запуска в контейнере
- `docs/bison-flex-install.md` — установка инструментов

---

## Сборка и запуск

```bash
cmake -B build
cmake --build build
```

Запуск на Linux/WSL:

```bash
./build/main ./examples/valid/test.mini
```

Запуск на Windows:

```powershell
.\build\Debug\main.exe .\examples\valid\test.mini
```
